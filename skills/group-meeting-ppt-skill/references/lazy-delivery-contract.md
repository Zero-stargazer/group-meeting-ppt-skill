# 成品交付约定（内部）

本 Skill 的成品路径是“PDF → 可编辑 PPTX 初稿”，不是“PDF → 大纲 → 让用户自己做 PPT”。

## 交付边界

- PDF 存在时，先在内部提取研究问题、方法、关键图、图注、关键结论和证据边界。
- 按论文实际主图和汇报时长确定页数；不使用固定页数。
- 关键图必须来自论文 PDF，并以读得清的原图或面板级裁切直接放入 PPT；图号、页码或来源信息随图保留。不得把整页论文、正文段落或图注段落当作“图表”。
- 每张幻灯片只放面向组会听众的标题、事实、图表、必要的证据边界和来源。
- 输出一个可编辑 PPTX 与一页核对清单。核对清单只列可能影响事实准确性的高风险项。

## 禁止出现在成品幻灯片中的文字

- “这一页目的”
- “你应该讲”
- “建议画面”
- “讲稿提示”
- “放图位置”
- “请用户补充 / 请用户自己完成”
- 任意内部文件路径、Prompt、评分规则或制作步骤

## 自动化失败时的降级原则

- 无法可靠读取 PDF 时，明确交付“证据简报”，不得命名为“组会 PPT 成品”。
- 读到 PDF 但无法可靠定位、裁切并核对关键图时，交付“证据简报”，不得用整页截图或空白占位框伪装成可直接汇报的 PPT 成品。
- 不编造图表解释、统计数字、样本数、方法条件或因果结论。
